JavaScript 执行上下文

Created2022-09-03|Updated2022-09-03|前端学习JavaScript
|Post View:

执行上下文

执行上下文：评估和执行 JavaScript 代码环境的抽象概念。每当 JavaScript 代码在运行时，它都在执行上下文中运行。

执行上下文的类型

  • 全局执行上下文

    默认上下文，任何不在函数中的上下文都在全局执行上下文中。

    执行两件事：

    ​ 创建一个全局 window 对象，并设置 this 的值为这个全局对象。

    ​ 一个程序只会有一个全局执行上下文。

  • 函数执行上下文

    每当一个函数被调用的时候，都会为这个函数创建一个新的上下文。

    每个函数都会有自己的执行上下文（在函数被调用的时候被创建）。

  • Eval 函数执行上下文

    执行 evel 函数内部的代码也有自己的上下文。

执行栈

执行栈，其他语言中的调用栈。先进后出的数据结构，用来存储代码运行时候创建的所有执行上下文。

当 JavaScript 引擎第一次遇到编写的脚本时：

  • 创建一个全局上下文压入当前执行栈
  • 遇到一个函数调用时，为该函数创建一个新的执行上下文并压入栈的顶部
  • 引擎执行执行上下文中位于栈顶的函数。当函数执行完成后，该函数的执行上下文从栈顶弹出，控制流程到达当前执行栈中的下一个上下文。

创建执行上下文

创建上下文两个阶段：创建阶段、执行阶段

创建阶段

  • this 值的决定（this绑定）
  • 创建词法环境组件
  • 创建变量环境组件

this 绑定

全局执行上下文中：this 为 window 对象。

函数执行上下文中：this 的值取决于函数如何被调用的。

  • 引用对象调用：this 设置为该引用对象
  • 其他情况：被设置为全局对象或者undefined（严格模式）

词法环境

词法环境两个组件：环境记录器和外部环境的引用

  • 环境记录器：存储变量和函数申明的实际位置
  • 外部环境引用：可以访问其父级的词法作用域

词法环境的两种类型：

全局环境（全局执行上下文）：无外部环境引用

函数环境：用户定义的变量存储在环境记录器中。其外部环境引用可能是全局环境或者包含此函数的外部函数

变量环境

变量环境也是一个词法环境

不同：

ES6中词法环境用来存粗函数声明和变量（let const）

变量环境存储 var 变量绑定
